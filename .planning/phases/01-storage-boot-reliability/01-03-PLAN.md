<objective>
Surface a one-time, dismissible, localized, screen-reader-reachable notice when boot recovery actually dropped one or more alarms (so the user knows to re-create them) — and ONLY then. Purpose: Completes BOOT-04's "recovers to a safe default and is logged" with the user-visible signal for the one case that can cause a missed alarm. Depends on Plan 01's `SalvageReport` flag (the detection logic); runs in parallel with Plan 02 (no shared files).

- Logic in `lib/app.dart` to read `SalvageReport.alarmsWereLost`, show a `Semantics`-wrapped dismissible SnackBar via the existing `_messangerKey`, and clear the flag once shown.
</objective>
