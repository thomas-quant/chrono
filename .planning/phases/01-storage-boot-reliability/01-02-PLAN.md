<objective>
Fix the boot black-screen / splash-hang epic: guard the boot isolate so it never touches credential-encrypted storage before unlock (defer-until-unlock, D-07), time-box `main()` init so a slow/failed recovery always degrades to the normal UI (D-06), and funnel both the boot path and app launch through the existing idempotent reschedule primitive so a reboot+unlock re-arms every alarm/timer exactly once (D-08).

Purpose: This is the core reliability fix. It depends on Plan 01's hardened loads — the time-boxed/guarded paths only degrade gracefully because corrupt/partial state now recovers instead of throwing.

Output:
- `lib/system/logic/device_lock.dart` — `isDeviceLocked()` probe (API-gated, no-op < API 24).
- Guard + try/catch fix in `handle_boot.dart` (BOOT-01 / BOOT-02 / D-07).
- Time-boxed init in `main.dart` that always reaches `runApp` (BOOT-01 / D-06).
- Optional native `isUserUnlocked` MethodChannel in `MainActivity.kt` and/or manifest narrowing (BOOT-02 mechanism / defense-in-depth) — chosen during the task per the boot-isolate reachability decision.
- A blocking human-verify checkpoint for the reboot-before-unlock validation that can only be confirmed on a secure-lock FBE device.
</objective>
