<objective>
Harden the existing plain-text-JSON storage path (Tier 1, per D-01 — no rewrite) so that interrupted writes can never corrupt a file, every JSON decode is guarded, and a corrupt list loses at most the bad entries instead of crashing or hanging. Also create the module-level "alarms were lost" flag that the boot path and UI notice (later plans) read.

Purpose: This is the foundational layer. The boot guard, time-boxed splash, and idempotent reschedule funnel (Plan 02) all depend on loads that recover instead of throw, and the one-time notice (Plan 03) reads the salvage flag created here. Storage primitives land first and independently.

Output:
- Atomic temp-write + rename in `saveTextFile` and `saveRingtone` (STOR-01 / D-02).
- Per-entry salvage in `listFromString`; `loadList` wraps it and never throws (BOOT-04 / D-04).
- Null-safe `SettingGroup.load()` that keeps the GetStorage fallback (STOR-02 / D-03 / D-05).
- New `lib/common/logic/salvage_report.dart` flag, set only on Alarm loss (D-06).
- Unit tests for atomic write, per-entry salvage, and the flag.
</objective>
